## Why This Revision

Phase 1 originally planned full Article CRUD via web UI (create/edit/delete forms + server actions).

After reviewing the actual workflow, this approach was revised:

- The web UI serves **readers**, not the author
- **Claude is the write path** — Dawn provides source material (URL or .md), Claude processes and persists it to the DB
- The DB remains the permanent store (not hardcoded data), enabling future search, filtering, and querying

Full reasoning: [ADR-012](../decisions/012-claude-managed-content-pipeline.md)

## Content Pipeline Workflow

```
Dawn provides one of:
  ├── YouTube URL     → Claude fetches title, description, summary
  ├── Blog/Article URL → Claude fetches and parses body content
  └── .md file        → Claude reads and structures content

Claude then:
  1. Analyzes and summarizes the source material
  2. Assigns category and tags
  3. Generates structured Markdown body
  4. Inserts into Supabase DB via Drizzle ORM:
     - articles table (title, content, summary, source_url, source_type, ...)
     - tags table (upsert)
     - article_tags table (relation)
```

## Updated DB Schema

### articles table (additions)

| Column | Type | Description |
|--------|------|-------------|
| `source_url` | `text` (nullable) | Original URL of the source content |
| `source_type` | `enum` | `youtube` \| `blog` \| `manual` |

### source_type values

| Value | Description |
|-------|-------------|
| `youtube` | Source is a YouTube video URL |
| `blog` | Source is a blog post or article URL |
| `manual` | Source is a manually written .md file |
