## Why This Revision

Phase 1.1's direct Claude-based URL analysis has quality and token cost limitations. NotebookLM MCP is introduced as a dedicated analysis layer to address both.

Full reasoning: [ADR-016](../decisions/016-notebooklm-analysis-layer.md)

---

## Proposed Workflow

```
User provides a URL (YouTube or blog) to Claude

Claude:
  1. Calls NotebookLM MCP with the URL
  2. NotebookLM analyzes the source (transcript, full body, etc.)
  3. Returns structured analysis (summary, key points, insights)

Claude:
  4. Formats the analysis as a structured .md file
  5. Saves the .md file to ~/Project/bnivibe/summaries/<topic>/
  6. Invokes the existing /dawnbase-post-article skill with the saved .md file
  7. Article is created in the DB and confirmed to the user
```
